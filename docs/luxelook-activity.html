Component View

Activity View: Event Recommendation

Authentication

  1. Browser loads the Next.js app shell.
  2. useAuth restores the cached JWT when possible.
  3. Frontend calls FastAPI auth routes.
  4. Backend uses mock auth or Supabase Auth by environment.
  5. JWT is returned and the session becomes authenticated.

Wardrobe Upload

  1. User drops an image into Wardrobe.
  2. Frontend calls /clothing/tag-preview.
  3. Backend tags category, color, season, formality, and descriptors.
  4. Duplicate detection runs before save.
  5. Category changes rerun preview tagging before upload.
  6. Final upload persists the item and starts media processing.

Discover / The Edit

  1. Backend builds a taste-seeded Discover feed.
  2. Pexels images are fetched and filtered to usable fashion results.
  3. Style tags are extracted and cached.
  4. User swipes like, love, or dislike.
  5. Raw interactions are stored.
  6. Learned preferences refresh into user_style_preferences.

Event + Beyond Your Wardrobe

  1. EventBrief plus raw_text_json is stored.
  2. All brief fields become event tokens.
  3. User-selected dress code overrides weaker inferred formality.
  4. Wardrobe combinations are scored and returned.
  5. LLM style directions add broader editorial options.
  6. Wearable pieces can receive external reference imagery.

Style Item

  1. User picks one anchor wardrobe item.
  2. The same structured brief editor is used as Event.
  3. The recommender tries to build around that anchor.
  4. If the wardrobe cannot fully support the look, editorial direction fills the gap.

Archive

  1. Archive loads stored events and stored suggestions.
  2. Structured event JSON is rebuilt into readable event summaries.
  3. Users revisit, compare, and rate historical looks.
  4. Existing combo ratings persist when the same look reappears.

Ratings, Skips, and Refresh

  1. Users can rate looks from 1 to 5.
  2. 0 is valid for None of these looks.
  3. Refresh avoids exact duplicate combos when better alternatives exist.
  4. Stored combo ratings carry forward to recurring looks.

Profile + AI Profiling

  1. Users save body details and styling traits.
  2. Visible avatar and AI profiling photo remain separate.
  3. AI can suggest face shape, body type, complexion, and hair traits.
  4. Saved profile traits feed future recommendation logic and Discover seeding.

Guide

  1. The Guide is a frontend-only education surface.
  2. It explains dress codes, seasons, descriptor families, and profile usage.
  3. It helps users correct AI tags with clearer fashion vocabulary.

Event Scenario Tester

  1. Tester loads saved EventBrief JSON scenarios.
  2. It uses the same shared form as the main Event page.
  3. It calls the real recommendation flow.
  4. Results render below, including style-direction moodboards.

What Changed Since the Old PDF

  • The current app uses /event and /archive, not legacy /events and /outfits.
  • The app now includes a Guide page and a Style Item page.
  • The Event flow is structured around EventBrief + raw_text_json.
  • Beyond your wardrobe now has visual moodboards and image enrichment.
  • Discover preference reliability and auto-refresh behavior are newer than the old PDF.
  • The isolated /test/event-scenarios surface did not exist in the old document.
